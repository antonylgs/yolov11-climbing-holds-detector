# Learning notes — Issue 001: Scaffold + pretrained smoke test

What was built, why each piece exists, and how to review it yourself.

## What this issue accomplished

The "tracer bullet": prove the entire stack works before writing any real feature code — environment → model download → inference → visible output. Every later issue builds on this skeleton.

```
pyproject.toml + uv.lock        the environment, pinned & reproducible
.gitignore                      keeps big binaries (data, weights) out of git
src/holds_detector/             empty package; real code lands in issue 002
scripts/generate_fixture.py     creates the synthetic test wall image
tests/fixtures/sample_wall.jpg  the test image (checked in, ~100 KB)
tests/test_smoke.py             the proof everything works
configs/ notebooks/ data/ models/   PRD §5 layout, filled by later issues
```

## Key decisions & why

### 1. `uv` instead of pip/venv/conda
`uv sync` reads `pyproject.toml`, creates `.venv/`, installs exact versions from `uv.lock`. Anyone (you, Colab, an agent) gets the *identical* environment from a clean clone. The lockfile is the reproducibility guarantee — that's why it's committed.

### 2. A *synthetic* fixture image, generated by a seeded script
- No licensing questions (a downloaded gym photo has copyright).
- Tiny and deterministic: `np.random.default_rng(42)` means re-running `scripts/generate_fixture.py` reproduces the byte-identical wall. Same trick the dataset split will use later (PRD: "seeded").
- It deliberately *looks* like a wall: grey textured background, t-nut bolt grid, irregular colored polygons with highlight/shadow — so later issues (color extraction!) can reuse it.

### 3. The smoke test asserts the machinery, not the predictions
`yolo11n-seg.pt` is pretrained on COCO (people, cars, dogs…). It has never seen a climbing hold, so detecting **zero objects on our wall is a pass, not a failure**. The test asserts:
- model loads (auto-downloads to `models/` on first run)
- inference runs on CPU and returns one well-formed `Results` object
- an annotated image gets written to a gitignored path (`data/outputs/smoke_annotated.jpg`)

This is the difference between "test the plumbing" and "test the model" — model quality gets its own issue (006, eval).

### 4. Why `n` (nano) here when the PRD says `s` (small)?
Smoke test only cares that segmentation *runs*; nano is the smallest/fastest download. Fine-tuning (issue 004/005) will use `yolo11s-seg` per the PRD.

### 5. `.gitignore` pattern worth knowing
```
data/*
!data/.gitkeep
```
Ignore everything *inside* `data/` but keep the directory itself in git via an empty `.gitkeep`. Git can't track empty directories — this is the standard workaround. Also `*.pt` globally: model weights never belong in a code repo (they're 6 MB–200 MB binaries; that's what `models/` + Drive/Roboflow are for).

### 6. Tooling wired in `pyproject.toml`
- **ruff** = linter (style + bug-prone patterns). Rules chosen: `E,F` (pycodestyle/pyflakes basics), `I` (import sorting), `UP` (modern syntax), `B` (bugbear — real bug catchers).
- **pytest** pointed at `tests/` via `[tool.pytest.ini_options]`.
- `[dependency-groups] dev` = deps only developers need (pytest, ruff), kept separate from runtime deps (ultralytics, opencv, numpy).

## Review it yourself

```bash
uv sync                                  # exact env from lockfile
uv run pytest -v                         # 2 tests pass (~25 s on CPU)
uv run ruff check .                      # lint clean
open tests/fixtures/sample_wall.jpg      # the synthetic wall
open data/outputs/smoke_annotated.jpg    # what YOLO drew (likely nothing — expected!)
uv run python scripts/generate_fixture.py  # regenerate fixture, byte-identical
```

Things worth eyeballing in the code:
- `tests/test_smoke.py` — note the docstring explaining *what a pass means*
- `scripts/generate_fixture.py` — seeded RNG usage; how the polygons are built from angles+radii

## Concepts you can now check off (cf. LESSON.md)

- **Pretrained weights**: `yolo11n-seg.pt` is "a model that already knows how to see" — we just ran it as-is. Fine-tuning (LESSON §2) starts from exactly this file format.
- **Instance segmentation output**: the `Results` object carries `.boxes` *and* `.masks` — boxes for "where", masks for "exact outline" (LESSON §3).
- **imgsz**: we ran at 640 here (speed; smoke only). Training/inference for real will use 1280 (LESSON §8, PRD §4).

## Questions to test yourself

1. Why is zero detections a *passing* smoke test? What would make it fail?
2. Why commit `uv.lock` but gitignore `models/yolo11n-seg.pt`?
3. If you deleted `.venv/` and `models/` right now, what two commands restore a working setup? (Answer: `uv sync`, `uv run pytest` — the model re-downloads itself.)
